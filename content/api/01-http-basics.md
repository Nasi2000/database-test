## HTTP-методы

| Метод | Назначение | Идемпотентен? | Пример |
|-------|-----------|---------------|--------|
| **GET** | Получить данные | Да | Список товаров |
| **POST** | Создать ресурс | Нет | Создать заказ |
| **PUT** | Полная замена | Да | Обновить профиль целиком |
| **PATCH** | Частичное обновление | Нет* | Сменить статус заказа |
| **DELETE** | Удалить | Да | Удалить черновик |

## Коды ответа (главные для SA)

### 2xx — успех
- **200 OK** — запрос выполнен, тело ответа есть
- **201 Created** — ресурс создан (обычно POST)
- **204 No Content** — успех без тела (DELETE)

### 4xx — ошибка клиента
- **400 Bad Request** — невалидные данные
- **401 Unauthorized** — не авторизован
- **403 Forbidden** — нет прав
- **404 Not Found** — ресурс не найден
- **409 Conflict** — конфликт (например, нет товара на складе)

### 5xx — ошибка сервера
- **500 Internal Server Error** — что-то сломалось на сервере
- **503 Service Unavailable** — сервис временно недоступен

## Структура HTTP-запроса

```
GET /api/v1/products?category=Ноутбуки HTTP/1.1
Host: localhost:3000
Accept: application/json
```

## Структура ответа TechStore

```json
{
  "data": [ ... ],
  "meta": { "total": 3, "limit": 50 }
}
```

## Практика

Откройте **API-песочницу** и выполните GET `/api/v1/products` — посмотрите статус 200 и структуру JSON.
