## Зачем очередь?

- Развязать системы по времени
- Сгладить пики нагрузки
- Гарантировать доставку (at-least-once)

## Базовые понятия

| Термин | Описание |
|--------|----------|
| **Producer** | Отправитель сообщения |
| **Consumer** | Получатель |
| **Topic / Queue** | Канал сообщений |
| **Offset** | Позиция чтения в потоке |
| **Dead Letter Queue** | «Кладбище» неудачных сообщений |

## RabbitMQ vs Kafka (упрощённо)

| | RabbitMQ | Kafka |
|---|----------|-------|
| Модель | Очередь, routing | Лог событий |
| Потребители | Конкурируют за сообщение | Каждый читает свой offset |
| Хранение | После ack удаляется | Retention по времени |
| Типичный кейс | Задачи, RPC-async | Event streaming, аналитика |

## Контракт события (пример)

```json
{
  "event_type": "order.paid",
  "event_id": "uuid",
  "occurred_at": "2024-09-01T12:00:00Z",
  "payload": {
    "order_id": 4,
    "customer_id": 3,
    "amount": "54990.00"
  }
}
```
