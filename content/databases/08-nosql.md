## Семейства NoSQL

| Тип | Примеры | Когда использовать |
|-----|---------|-------------------|
| Document | MongoDB | Гибкая схема, JSON-документы |
| Key-Value | Redis | Кэш, сессии, очереди |
| Column | Cassandra | Большие объёмы записи, time-series |
| Graph | Neo4j | Сложные связи (соцсети, рекомендации) |

## SQL vs NoSQL — для аналитика

| Критерий | SQL | NoSQL |
|----------|-----|-------|
| Схема | Жёсткая, заранее | Гибкая / schema-on-read |
| Связи | JOIN, FK | Вложенность или денормализация |
| Транзакции | ACID из коробки | Зависит от БД |
| Масштабирование | Вертикальное + реплики | Горизонтальное шардирование |

## Когда оставаться на SQL

- Финансы, заказы, инвентарь — нужна целостность
- Сложные отчёты с JOIN
- Команда знает SQL

## Когда смотреть на NoSQL

- Каталог с разной структурой товаров
- Логи, события, метрики (append-only)
- Кэш и real-time (Redis)
- Очень высокая нагрузка на запись

## Polyglot Persistence

В одном проекте часто **несколько** хранилищ:
- PostgreSQL — заказы и клиенты
- Redis — корзина и сессии
- Elasticsearch — поиск по каталогу

Задача аналитика — описать, **какие данные где** живут и как синхронизируются.
