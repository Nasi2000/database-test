## Что такое индекс?

Структура данных (часто B-tree), ускоряющая поиск по столбцу. Аналог оглавления в книге.

## Когда индекс нужен

- Частые фильтры: `WHERE customer_id = ?`
- JOIN по FK
- Сортировка по столбцу на больших таблицах
- UNIQUE-ограничения

## Пример из TechStore

```sql
CREATE INDEX idx_orders_customer ON orders(customer_id);
CREATE INDEX idx_orders_status ON orders(status);
```

## Когда индекс не нужен

- Маленькие таблицы (< 1000 строк)
- Столбец с низкой селективностью (пол «пол» в таблице людей)
- Таблицы с частыми массовыми INSERT

## EXPLAIN — инструмент аналитика

```sql
EXPLAIN ANALYZE
SELECT * FROM orders WHERE customer_id = 1;
```

Смотрите на `Seq Scan` (полный перебор) vs `Index Scan`.

## Компромиссы

| Плюс | Минус |
|------|-------|
| Быстрее SELECT | Медленнее INSERT/UPDATE |
| Быстрее JOIN | Дополнительное место на диске |

## Что должен знать SA

- Не проектировать запросы «на все поля сразу»
- Понимать, что N+1 запросов — проблема API, не только БД
- На ревью схемы спрашивать про ожидаемые объёмы и паттерны чтения
