## Транзакция

Группа операций, которая выполняется **целиком** или **не выполняется вообще**.

```sql
BEGIN;
  UPDATE products SET stock_qty = stock_qty - 1 WHERE id = 4;
  INSERT INTO order_items (order_id, product_id, quantity, unit_price)
  VALUES (9, 4, 1, 89990.00);
  UPDATE orders SET total_amount = total_amount + 89990 WHERE id = 9;
COMMIT;
-- или ROLLBACK; при ошибке
```

## ACID

| Свойство | Смысл |
|----------|-------|
| **A**tomicity | Всё или ничего |
| **C**onsistency | Бизнес-правила не нарушаются |
| **I**solation | Параллельные транзакции не мешают друг другу |
| **D**urability | После COMMIT данные сохранены |

## Пример для аналитика: оформление заказа

**Требование:** при оформлении заказа одновременно:
1. Создаётся заказ
2. Списывается остаток
3. Создаются позиции

Если шаг 3 упал — заказ и остаток не должны «зависнуть» в промежуточном состоянии.

## Уровни изоляции (кратко)

- **Read Uncommitted** — грязное чтение (редко используют)
- **Read Committed** — default в PostgreSQL
- **Repeatable Read** / **Serializable** — строже, меньше аномалий

## Блокировки и deadlock

Две транзакции ждут друг друга → deadlock. PostgreSQL откатывает одну из них.
