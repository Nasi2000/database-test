## Что такое реляционная БД?

Данные хранятся в **таблицах** (отношениях). Связи между таблицами задаются через **ключи**.

## Базовые понятия

| Термин | Описание |
|--------|----------|
| **Таблица** | Набор строк одного типа сущности |
| **Строка (record)** | Один экземпляр сущности |
| **Столбец (column)** | Атрибут с типом данных |
| **Primary Key (PK)** | Уникальный идентификатор строки |
| **Foreign Key (FK)** | Ссылка на PK другой таблицы |

## Пример: TechStore

```
customers          orders              order_items
─────────          ──────              ───────────
id (PK)            id (PK)             id (PK)
email              customer_id (FK)──►   order_id (FK)──►
full_name          status                product_id (FK)──►
city               total_amount          quantity
                   created_at            unit_price
```

## Типы связей

| Связь | Пример | Реализация |
|-------|--------|------------|
| 1:1 | Паспорт ↔ Человек | FK с UNIQUE |
| 1:N | Клиент → Заказы | FK в таблице «многих» |
| N:M | Студенты ↔ Курсы | Промежуточная таблица |

## Зачем аналитику это знать?

- Проектировать модель данных под требования
- Понимать ограничения системы («нельзя удалить клиента с заказами»)
- Писать осмысленные SQL-запросы для проверки данных
- Общий язык с разработчиками и DBA

## Практика

Откройте раздел **SQL-песочница** и выполните:

```sql
SELECT table_name FROM information_schema.tables
WHERE table_schema = 'public';
```

Изучите структуру таблиц TechStore.
