## Типы требований

### Функциональные (ЧТО система делает)
- Пользователь может оформить заказ
- Система отправляет email при смене статуса

### Нефункциональные (КАК система работает)
- Время ответа API < 200 мс
- Доступность 99.9%
- Поддержка 10 000 одновременных пользователей

## Методы сбора

| Метод | Когда использовать |
|-------|-------------------|
| Интервью | Глубокое понимание одной роли |
| Воркшоп | Согласование между стейкхолдерами |
| Наблюдение | Когда люди делают не то, что говорят |
| Анализ документов | Есть регламенты, договоры, legacy-система |
| Прототип | Быстрая валидация идей с UI |

## Вопросы, которые стоит задавать

1. **Кто** пользователь? Какая у него роль?
2. **Зачем** ему эта функция? Какую проблему решаем?
3. **Что** происходит сейчас без системы?
4. **Когда** и при каких условиях срабатывает сценарий?
5. **Что** считается ошибкой / исключением?

## Типичные ошибки

- Фиксировать решение вместо проблемы («нужна кнопка» вместо «нужно быстро повторить заказ»)
- Не уточнять граничные случаи (пустая корзина, нулевая цена)
- Игнорировать нефункциональные требования
