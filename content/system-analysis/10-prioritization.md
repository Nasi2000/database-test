## Зачем приоритизировать?

Ресурсы ограничены. Аналитик помогает команде делать **сначала самое ценное**.

## MoSCoW

| Категория | Смысл |
|-----------|-------|
| **Must** | Без этого релиз невозможен |
| **Should** | Важно, но можно отложить |
| **Could** | Приятно иметь |
| **Won't** | Сознательно не в этом релизе |

**Пример TechStore:** Must — оформление заказа; Should — история заказов; Could — рекомендации товаров.

## RICE

```
RICE = (Reach × Impact × Confidence) / Effort
```

| Фактор | Вопрос |
|--------|--------|
| Reach | Сколько пользователей затронет за период? |
| Impact | Насколько сильно изменит опыт (0.25–3)? |
| Confidence | Насколько уверены в оценке (%) |
| Effort | Человеко-месяцы |

## Value vs Effort (матрица)

```
        Высокая ценность
              │
   Quick wins │ Big bets
──────────────┼──────────────
   Fill-ins   │ Money pit
              │
        Низкая ценность
    Малые усилия → Большие усилия
```

## Как аналитик участвует

1. Формулирует ценность в терминах пользователя
2. Декомпозирует крупные фичи на оцениваемые части
3. Выявляет зависимости (без API оплаты нельзя запустить checkout)
4. Документирует допущения в оценке
