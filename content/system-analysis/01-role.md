## Кто такой системный аналитик?

**Системный аналитик (SA)** — специалист, который переводит бизнес-потребности в понятные для команды разработки требования. Он — мост между заказчиком и IT.

## Основные задачи

| Задача | Пример |
|--------|--------|
| Сбор требований | Интервью с владельцем продукта, анализ текущих процессов |
| Анализ и формализация | User stories, use cases, диаграммы |
| Проектирование решения | Модель данных, API, интеграции |
| Согласование | Ревью с архитектором, разработчиками, тестировщиками |
| Сопровождение | Уточнения в спринте, приёмка результата |

## Ключевые навыки

1. **Коммуникация** — умение задавать правильные вопросы и фиксировать договорённости
2. **Системное мышление** — видеть связи между процессами, данными и системами
3. **Документирование** — писать так, чтобы не было двусмысленности
4. **Базы данных и SQL** — понимать, как хранятся данные (не обязательно писать код)
5. **Базовое понимание API** — REST, форматы JSON, коды ответов

## Артефакты аналитика

- Vision / Scope документ
- User Stories и Acceptance Criteria
- Диаграммы (BPMN, UML, ER)
- Спецификация API (OpenAPI)
- Модель данных (логическая/физическая схема)
